# Code of Conduct

## Our Commitment

We want `superpowers-openspec-team-skills` to be a respectful, constructive, and low-drama project for maintainers, contributors, and users.

Examples of behavior that contribute to a positive environment include:

- being respectful in disagreement
- assuming good intent while still giving direct technical feedback
- focusing discussion on repository behavior, evidence, and user impact
- helping newcomers understand repository structure and workflow boundaries
- documenting risks and tradeoffs clearly

Examples of unacceptable behavior include:

- personal attacks, insults, or harassment
- hostile or dismissive communication
- bad-faith arguments or deliberate misrepresentation
- publishing private information without permission
- pressuring maintainers to merge unsafe automation or trust-boundary changes without review

## Repository-Specific Expectations

This repository includes workflow and memory tooling that can change how AI assistants behave. That makes review tone and technical rigor especially important.

Contributors should:

- be explicit about whether a change is opt-in or automatic
- avoid overstating cross-platform support or verification status
- call out safety, trust, or automation boundary changes clearly
- prefer evidence over assumption when describing behavior

## Enforcement Responsibilities

Project maintainers are responsible for clarifying and enforcing this Code of Conduct. They may remove, edit, or reject contributions, comments, issues, or pull requests that do not align with this policy.

## Scope

This Code of Conduct applies within:

- repository issues
- pull requests
- discussions tied to this repository
- review comments and contribution conversations

## Reporting

To report unacceptable behavior or a serious interaction concern, contact the maintainers through the security and contact guidance in [SECURITY.md](/D:/spring_AI/superpowers-openspec-team-skills/SECURITY.md).

## Consequences

Maintainers may take actions including:

- asking for a conversation reset or edit
- locking or closing threads
- rejecting contributions
- temporarily or permanently restricting participation

## Attribution

This Code of Conduct is adapted from the spirit of the [Contributor Covenant](https://www.contributor-covenant.org/), simplified for this repository.
